Modify Rasters to Meet Desired Projections and Resolutions.

The raster objects you loaded and their descriptions are shown below. Each has a different class, resolution, extent, and other spatial characteristics. All need to match ext.rast (ie. be identical in projection, extent, resolution) or analytical operations, such as the calculation of descriptive statistics (eg measures of central tendency, dispersion) or complex statistical modelling, will be difficult.
Object Name Description
* ext.rast # the base raster ALL rasters must match
* elev1km.wgs84 # elev 1km raster, WGS84
* ndvi1500m.wgs84 # NDVI raster at 500m resolution, WGS84
* precdry1km.wgs84 # precip at 1km raster, WGS84
* rough1km.wgs84 # terrain roughness 1km raster
* cp.nlcd30m.aea # NLCD at 30m raster, AEA
* taveYrAvg1km.wgs84 # avg annual temp 1km raster, WGS84

If the analytical goal is to simply ensure all rasters have identical spatial characteristics, the simplest approach would be run all the rasters through a loop or function and modify all to the identified base raster (here, ext.rast). However, for illustrative purposes we will examine and modify each to the base raster so that the process of modification can be better understood.

The projectRaster() call (package raster) is the easiest, most repeatable method to ensure rasters are all identical. As noted above, a base raster is required. Depending on whether the underlying values of the raster are categorical or numerical in scale, either a nearest neighbor (ngb) for the categorical scale or a bilinear for the numerical scale data can be applied. These are implemented using the method = option, with bilinear being the default.

Let''s start with elevation data.

##   elev1km.wgs84 vs. ext.rast as base raster; NOTE <1 min runtime
   ext.rast;  elev1km.wgs84   # examine; NOTE wrong extent, dimensions
   cp.elev <- projectRaster(elev1km.wgs84, ext.rast)
   ext.rast;  cp.elev  # examine; now identical ??
   names(cp.elev) <- "elev";  cp.elev  # apply rename option

Creating a Stack of Rasters

One of the principal reasons for ensuring common spatial characteristics among rasters is so they can be “stacked.” A raster stack is simply that - individual layers stacked on top of one another in a single raster object. A raster stack is created using the stack() call.

One obvious, and common, use of a raster stack is to extract data, where a data matrix of [X,Y] is “punched” through the stack and the associated values from all individual layers in the stack are tagged to each [X,Y]. Creation of the stack is the first step in this process..

## create a stack of rasters of same projection, extent, resolution, dimension
   cp.allvars <- stack(cp.elev, cp.ndvi, cp.nlcd)
   cp.allvars  # examine

Creating a Raster Brick

An analogue of the stack is the raster brick, created using the brick() call. Like a stack, a brick is a multiple layer raster object. It is more commonly used in multi-band data, such as Landsat or other remote sensors. Processing time can be more rapid with a brick than a stack.

## create a brick of rasters
   cp.brick <- brick(cp.elev, cp.ndvi, cp.nlcd)
   cp.brick
